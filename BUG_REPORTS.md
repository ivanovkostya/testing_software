## БАГ-РЕПОРТ №BUG-001

| Атрибут | Значение |
|---------|----------|
| **Уникальный идентификатор** | BUG-001 |
| **Название** | Поле «Сумма перевода» принимает отрицательные значения |
| **Серьёзность** | Major |
| **Приоритет** | High |
| **Статус** | Open |
| **Автор** | Тестировщик |
| **Дата обнаружения** | 2026-04-27 |

### Предусловия
1. Приложение запущено по адресу `http://localhost:8000`
2. Выбрана валюта «Рубли»

### Окружение

| Параметр | Значение |
|----------|----------|
| ОС | Windows 10 / Ubuntu 22.04 |
| Браузер | Google Chrome 120+ |
| Устройство | ПК |

### Шаги воспроизведения
1. Открыть главную страницу приложения
2. Выбрать валюту «Рубли»
3. В поле «Сумма перевода» ввести `-500`
4. Нажать кнопку «Перевести»

### Фактический результат
Кнопка «Перевести» активна. При нажатии появляется сообщение: «Перевод -500 принят банком!»

### Ожидаемый результат
Поле не должно принимать отрицательные значения. Кнопка «Перевести» должна быть неактивна. При попытке ввода минуса сумма должна становиться положительной либо отображаться ошибка валидации.

### Дополнительная информация
Проблема воспроизводится на всех валютах.

### Связанные тесты
- ТК-10
- `test_10_negative_amount_input`

## БАГ-РЕПОРТ №BUG-002

| Атрибут | Значение |
|---------|----------|
| **Уникальный идентификатор** | BUG-002 |
| **Название** | Кнопка «Перевести» неактивна при достаточном балансе (balance=110, amount=100) |
| **Серьёзность** | Critical |
| **Приоритет** | High |
| **Статус** | Open |
| **Автор** | Тестировщик |
| **Дата обнаружения** | 2026-04-27 |

### Предусловия
1. Приложение запущено по адресу `http://localhost:8000?balance=110&reserved=0`
2. Выбрана валюта «Рубли»

### Окружение

| Параметр | Значение |
|----------|----------|
| ОС | Windows 10 / Ubuntu 22.04 |
| Браузер | Google Chrome 120+ |
| Устройство | ПК |

### Шаги воспроизведения
1. Открыть страницу с параметром `?balance=110&reserved=0`
2. Выбрать валюту «Рубли»
3. В поле «Сумма перевода» ввести `100`
4. Проверить состояние кнопки «Перевести»

### Фактический результат
Кнопка «Перевести» неактивна. Отображается сообщение «Недостаточно средств на счете».

### Ожидаемый результат
Кнопка должна быть активна, так как баланс (110) позволяет выполнить перевод с учётом комиссии (100 + 10 = 110).

### Дополнительная информация
Баланс равен сумме перевода + комиссия, но приложение считает это ошибкой.

## БАГ-РЕПОРТ №BUG-003

| Атрибут | Значение |
|---------|----------|
| **Уникальный идентификатор** | BUG-003 |
| **Название** | При резерве, превышающем баланс, отображается отрицательный доступный остаток |
| **Серьёзность** | Major |
| **Приоритет** | Medium |
| **Статус** | Open |
| **Автор** | Тестировщик |
| **Дата обнаружения** | 2026-04-27 |

### Предусловия
1. Приложение запущено по адресу `http://localhost:8000?balance=10000&reserved=15000`
2. Выбрана валюта «Рубли»

### Окружение

| Параметр | Значение |
|----------|----------|
| ОС | Windows 10 / Ubuntu 22.04 |
| Браузер | Google Chrome 120+ |
| Устройство | ПК |

### Шаги воспроизведения
1. Открыть страницу с параметрами `?balance=10000&reserved=15000`
2. Выбрать валюту «Рубли»
3. Посмотреть на отображение баланса

### Фактический результат
Баланс отображается как `10'000`, резерв как `15'000`. Доступный остаток (баланс - резерв = -5'000) явно не отображается, но при попытке перевода кнопка неактивна, и нет понятного сообщения о том, что резерв превышает баланс.

### Ожидаемый результат
Приложение должно отображать сообщение о том, что резерв превышает баланс, и скрыть возможность перевода. Доступный остаток не должен быть отрицательным в интерфейсе.

### Дополнительная информация
Пользователь видит положительный баланс, но не может сделать перевод без объяснения причины.

### Связанные тесты
- ТК-07
- `test_07_reserved_more_than_balance`

## БАГ-РЕПОРТ №BUG-005

| **Название** | При передаче нечисловых значений в URL отображается `NaN` в интерфейсе |
| **Серьёзность** | Minor |
| **Приоритет** | Low |
| **Статус** | Open |
| **Автор** | Тестировщик |
| **Дата обнаружения** | 2026-04-27 |

### Предусловия
1. Приложение запущено по адресу `http://localhost:8000?balance=abc&reserved=xyz`

### Окружение

| Параметр | Значение |
|----------|----------|
| ОС | Windows 10 / Ubuntu 22.04 |
| Браузер | Google Chrome 120+ |
| Устройство | ПК |

### Шаги воспроизведения
1. Открыть страницу с параметрами `?balance=abc&reserved=xyz`
2. Выбрать валюту «Рубли»
3. Проверить отображение баланса и резерва

### Фактический результат
В интерфейсе отображается:
- «На счету: **NaN** ₽»

Приложение должно корректно обрабатывать нечисловые значения в URL-параметрах. `NaN` (Not a Number) не должен быть виден конечному пользователю. Некорректные значения следует заменять на `0` или отображать прочерк.

### Дополнительная информация
Проблема воспроизводится с любыми нечисловыми значениями (`abc`, `xyz`, `hello`, `!@#` и т.д.).
